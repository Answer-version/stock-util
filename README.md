# QuantVibe 中文因子选股面板

一个面向真实行情的中文可视化量化研究工具。当前版本聚焦加密货币日线场景，用技术因子做横截面打分，按 `Top N` 方式构建组合，并用图形化页面展示回测结果。

本系统不使用演示数据，默认直接拉取真实行情。主数据源为 `ccxt`，日线场景下内置了 `OKX` 备用接口。

## 1. 系统能做什么

- 输入一组标的，例如 `BTC/USDT, ETH/USDT`
- 选择回测时间区间
- 选择持仓数量 `Top N`
- 用多个技术因子给标的打分
- 按“收盘生成信号，下一根开盘成交”规则回测
- 在中文可视化页面查看收益、回撤、持仓、得分和原始数据
- 导出 `CSV / PNG / HTML` 报告

## 2. 安装与启动

### 2.1 安装依赖

如果你还没有虚拟环境，可以先创建一个：



## 4. 页面怎么使用

页面左侧是参数区，右侧是结果区。

### 4.0 一次完整使用流程

如果你想最快上手，可以直接按这 7 步操作：

1. 打开页面 `http://127.0.0.1:8501/`
2. 保持默认参数不变
3. 点击 `运行可视化回测`
4. 先看 `总览` 页的 4 个核心指标
5. 再看 `因子视角` 和 `持仓明细`
6. 用 `最新持仓` 确认当前该买哪些标的
7. 需要保存结果时，点击页面上方的导出按钮

### 4.1 左侧参数区

#### 标的池

输入要研究的交易对，用英文逗号分隔，例如：

```text
BTC/USDT, ETH/USDT, SOL/USDT
```

建议：

- 首次使用控制在 `2-4` 个标的
- 先选自己熟悉的主流币
- 不要一开始就放太多标的，否则真实行情请求更慢，也更难分析结果

#### 开始日期 / 结束日期

表示回测区间。系统会在这个区间内拉取真实日线数据，计算因子并回测。

建议：

- 第一次可以先跑 `3-6` 个月
- 如果数据源偶发不稳定，先缩短时间范围再试

#### 持仓数量 Top N

表示每个调仓时点买入“综合得分最高”的前 `N` 个标的。

例如：

- `Top N = 1`：只买分数最高的 1 个
- `Top N = 2`：买分数最高的 2 个，等权持仓
- `Top N = 3`：买分数最高的 3 个，等权持仓

本系统当前的组合构建方式是：

- 先对每个标的计算综合得分
- 按分数从高到低排序
- 选前 `N` 个
- 入选标的等权分配

#### 启用因子

当前内置 5 个技术因子：

- `20日动量`：近 20 日涨幅，越强越容易得高分
- `20日波动率`：近 20 日波动率，系统里取了负号，越稳越容易得高分
- `RSI(14)`：14 日相对强弱指标
- `5/20日均线偏离`：短均线相对长均线的强弱程度
- `20日量比`：当前成交量相对近 20 日均量的强弱

这些因子会先做截面标准化，再合成为综合得分。

### 4.2 运行回测

参数配置完成后，点击：

```text
运行可视化回测
```

系统会执行以下流程：

1. 拉取真实行情
2. 计算每个因子
3. 生成综合得分
4. 构建 `Top N` 组合
5. 按下一根开盘成交规则回测
6. 展示图表和表格

## 5. 结果页怎么看

结果页分成几个标签页。建议按下面顺序看。

### 5.0 推荐阅读顺序

建议不要一上来就盯着净值曲线看，而是按下面顺序判断：

1. `总览`
2. `因子视角`
3. `持仓明细`
4. `市场视角`
5. `原始数据`

这样做的原因是：

- `总览` 先判断策略值不值得继续看
- `因子视角` 先看当前强弱排序是否清晰
- `持仓明细` 再确认系统当前实际要买谁
- `市场视角` 用来辅助理解走势和换手
- `原始数据` 用来做最终复核

### 5.1 总览

这是先判断“这套参数值不值得信”的第一站。

这里会显示 4 个核心指标：

- `总收益`
- `夏普比率`
- `最大回撤`
- `胜率`

#### 总收益

表示整个回测区间内，这套策略最后赚了多少。

怎么看：

- 越高越好
- 但不要只看这个指标
- 如果总收益高，但回撤很深、换手很高，实际不一定好用

#### 夏普比率

表示单位波动承担下，策略换来的收益质量。

粗略理解：

- 大于 `1`：通常说明风险收益质量开始比较像样
- 大于 `0`：说明有正向风险补偿
- 小于等于 `0`：说明这套参数大概率不值得继续用

#### 最大回撤

表示从历史高点往下掉得最深的一次幅度。

怎么看：

- 数值越接近 `0` 越好
- 如果你看到 `-30%`、`-40%` 甚至更深，说明过程会比较难熬
- 这个指标决定你是否“扛得住”，不是只看最后赚不赚钱

#### 胜率

表示样本期内，正收益周期占比。

怎么看：

- 高一些通常更舒服
- 但胜率不是越高越一定好
- 有些策略胜率一般，但单次盈利大，也可能整体赚钱

### 5.2 市场视角

这里主要看两张图：

- `价格相对强弱`
- `换手节奏`

#### 价格相对强弱

所有标的都从同一起点开始画线，方便比较谁更强。

怎么看：

- 如果某个标的整段时间明显更强，通常更有机会在综合得分里排前面
- 如果某个标的长期明显偏弱，它被选中的次数通常也会少

#### 换手节奏

看组合切换是否频繁。

怎么看：

- 换手过高，说明信号容易来回切换
- 换手高会带来更多交易成本
- 如果收益一般但换手很高，这套参数要谨慎

### 5.3 因子视角

这里的核心是 `综合得分热力图`。

你可以把它理解成：

- 横轴是时间
- 纵轴是标的
- 颜色越强，说明该标的在那个时点的综合得分越高

怎么看：

- 最近连续多天颜色更强的标的，通常是更稳定的候选
- 如果一个标的颜色忽强忽弱，说明信号不稳定
- 如果多个标的颜色接近，说明当前优势不明显，组合可能更容易轮动

这个页面适合回答的问题是：

- 最近是谁更强
- 当前领先是不是持续性的
- 这次轮动是不是很突然

### 5.4 持仓明细

这是最重要的落地页面。

这里主要看两块：

- `持仓轮动`
- `最新持仓`

#### 持仓轮动

用来观察系统过去一段时间到底在持有谁、切换频率高不高。

怎么看：

- 如果持仓切换很平滑，说明信号相对稳定
- 如果频繁来回切换，说明策略更敏感，实盘执行要更谨慎

#### 最新持仓

这个表是你当前最应该看的地方。

它直接回答：

- 当前系统最看好哪些标的
- 每个标的应该配多少仓位

最实用的理解方式：

- `1 个标的 + 100%`：当前就重点买这个标的
- `2 个标的 + 各 50%`：当前不是二选一，而是两个都买
- `3 个标的 + 各 33.33%`：说明系统认为当前强势标的分散，不应押单一标的

### 5.5 原始数据

这个标签页适合做复核，不是第一判断入口。

你可以在这里查看：

- 价格明细
- 得分明细

适合的场景：

- 想核对某个时间点的得分变化
- 想确认某个标的为什么被选中
- 想把原始数据导出后自己做二次分析

## 6. 如何判断要买哪个标的

这是最关键的部分。

### 6.1 先说结论

真正的“买入名单”，请以 `持仓明细` 页里的 `最新持仓` 表为准。

不要直接用：

- `总收益` 决定买谁
- 单看某一张价格图决定买谁
- 只凭感觉看哪个涨得快就买谁

系统的选标逻辑是：

1. 每个交易日对所有标的计算综合得分
2. 把综合得分从高到低排序
3. 选出前 `Top N` 个
4. 下一根开盘按等权买入

也就是说：

- `Top N = 1` 时：买 `最新持仓` 里唯一的那个标的
- `Top N = 2` 时：买 `最新持仓` 里前 2 个标的，各一半仓位
- `Top N = 3` 时：买 `最新持仓` 里前 3 个标的，等权配置

### 6.2 实际操作时怎么看

建议按这个顺序判断：

#### 第一步：先看“总览”是否合格

先判断这套参数有没有继续看的价值。

优先看：

- `总收益` 是否为正
- `夏普比率` 是否明显高于 0
- `最大回撤` 是否在你能接受的范围内
- `胜率` 是否太低

如果这几项整体很差，就不要急着根据这次结果下结论买入。

#### 第二步：看“因子视角”的热力图

热力图展示最近一段时间各标的的综合得分。

怎么看：

- 颜色更强、近期持续更高的标的，更可能排在前面
- 如果某个标的只是某一天突然冲高，但持续性差，要小心是假信号
- 如果一个标的最近连续多天得分领先，信号通常更稳定

#### 第三步：看“持仓明细”里的最新持仓

这里是最直接的买入参考。

重点看：

- 最新一期有哪些标的
- 每个标的权重是多少

例子：

- 如果 `最新持仓` 里只有 `BTC/USDT = 100%`，那当前结论就是优先买 `BTC/USDT`
- 如果最新持仓是 `BTC/USDT = 50%`、`ETH/USDT = 50%`，那就不是二选一，而是按系统逻辑两者都买

#### 第四步：再用“价格相对强弱”做交叉验证

如果 `最新持仓` 里的标的，同时在相对强弱图里也处于更强的走势，说明因子结果与市场表现比较一致。

这时信号通常更容易理解。

如果出现这种情况要谨慎：

- 热力图里刚升温
- 最新持仓刚切换
- 换手节奏又很高

这说明当前信号可能还不够稳定。

### 6.3 一个简单判断模板

你可以直接按下面这套规则使用：

1. 跑回测
2. 先看 `总收益 > 0`
3. 再看 `夏普比率 > 0`
4. 再看 `最大回撤` 是否在你的风险容忍范围内
5. 去看 `持仓明细 -> 最新持仓`
6. 以 `最新持仓` 作为当前要买的标的列表
7. 用 `因子热力图` 和 `价格相对强弱` 做确认

如果你只想得到“当前该买谁”这个答案：

- `Top N = 1`：买 `最新持仓` 里那个 100% 权重的标的
- `Top N > 1`：买 `最新持仓` 里所有标的，按权重分配

### 6.4 一套最稳的人工决策流程

如果你不想每次都临场判断，可以固定用这套流程：

#### 适合买入的情况

同时满足下面几条时，信号通常更值得参考：

- `总收益` 为正
- `夏普比率` 大于 `0`
- `最大回撤` 没有超出你的容忍范围
- `因子热力图` 里最近几天有持续领先的标的
- `最新持仓` 清晰，不是频繁切换后的第一天

#### 需要谨慎的情况

出现下面情况时，不建议直接照单买入：

- 总收益一般，但回撤很大
- 夏普很低甚至为负
- 换手明显过高
- 热力图里最近几天领先标的来回切换
- 最新持仓刚从一个标的瞬间切换到另一个标的

#### 最终买入原则

最后只用一句话收口：

- 看 `最新持仓` 决定买谁
- 看 `总览 + 热力图 + 换手` 决定这次信号值不值得信

## 7. 标准操作示例

下面给你一套后续可以反复照着跑的标准示例。

### 示例 1：判断当前只买一个标的

参数：

- 标的池：`BTC/USDT, ETH/USDT`
- 区间：`2025-01-01` 到 `2025-06-30`
- `Top N = 1`

操作：

1. 点击 `运行可视化回测`
2. 看 `总览` 是否整体为正向
3. 看 `因子视角` 最近谁更强
4. 打开 `持仓明细`
5. 看 `最新持仓`

结论方式：

- 如果 `最新持仓` 是 `BTC/USDT = 100%`
  当前按系统规则优先买 `BTC/USDT`
- 如果 `最新持仓` 是 `ETH/USDT = 100%`
  当前按系统规则优先买 `ETH/USDT`

### 示例 2：判断当前应该分散持有

参数：

- 标的池：`BTC/USDT, ETH/USDT, SOL/USDT`
- `Top N = 2`

结论方式：

- 如果 `最新持仓` 显示 `BTC/USDT = 50%`、`ETH/USDT = 50%`
  那么这次不是只买一个，而是按系统逻辑同时持有两个

适用理解：

- 当多个标的近期都比较强时，系统会用 `Top N` 做分散
- 这时不要强行主观挑一个，应该尊重系统的组合输出

### 示例 3：发现信号不稳时怎么处理

如果你看到下面这种组合：

- `总收益` 不高
- `夏普比率` 接近 0 或为负
- `最大回撤` 很深
- `换手节奏` 很高
- `最新持仓` 最近频繁切换

建议处理方式：

1. 先不要直接参考这次“买入名单”
2. 缩短标的池
3. 降低 `Top N`
4. 保留主流币重新跑一次

通常更稳的重跑方式是：

- 标的池收缩到 `BTC/USDT, ETH/USDT`
- `Top N` 先用 `1`
- 因子先保持默认全选

## 10. 常见问题

### 9.1 点击运行后提示行情源不可用

可以按下面顺序处理：

1. 缩短日期范围
2. 减少标的数量
3. 稍后重试

建议先从：

```text
BTC/USDT, ETH/USDT
```

开始。

### 9.2 为什么“当前该买谁”不是看总收益最高的币

因为系统不是比较“谁涨得最多”，而是在每个时点比较“谁当前综合得分更高”。

真正的买入结论来自：

- 最新一期综合得分排序
- `Top N` 规则
- `最新持仓` 表

### 9.3 为什么最新持仓会变

因为因子分数每天都会随着行情变化而变化。

所以系统选出来的标的不是永久不变的，而是会轮动。

- `dist/QuantVibe.app` 或 `dist/artifacts/QuantVibe-macOS.dmg`
- `dist/QuantVibe/` 或 `dist/artifacts/QuantVibe-windows.zip`

### 14.2 GitHub 自动构建

仓库已经带了 GitHub Actions 工作流：

- 推送到 `main` 时会自动构建
- 也可以手动触发 `Build Desktop Apps`

它会分别在：

- `macos-latest`
- `windows-latest`

上产出桌面包，并把结果上传为 Actions Artifacts。

### 14.3 打包注意事项

- Windows 版必须在 Windows 环境构建
- macOS 版必须在 macOS 环境构建
- 当前方案是“桌面壳 + 本地 Web 服务”，不是纯原生 UI
- 真实行情依赖网络连接，离线时无法正常拉取数据
- 如果要给外部用户分发，后续最好补 mac 签名公证和 Windows 签名
